# مرجع النظام الشامل

> وثيقة مرجعية رئيسية — اقرأها قبل أي تطوير جديد.

---

## 1. نظرة عامة

نظام عربي (RTL) لتسجيل حضور وانصراف الموظفين عبر التعرف على الوجه، مع لوحة تحكم إدارية وتقارير.

| الطبقة | التقنية |
|--------|---------|
| الواجهة | Next.js 14 App Router · React 18 · Tailwind · shadcn/ui |
| المصادقة | Supabase Auth (بريد + كلمة مرور) |
| قاعدة البيانات | PostgreSQL (Supabase) عبر Prisma 7 + `@prisma/adapter-pg` |
| التعرف على الوجه | @vladmandic/human — بصمة 1024-d (عميل) |
| النشر | Vercel (`fra1`) |

**جذر التطبيق:** `attendance-system/`  
**جذر المستودع:** `نظام حضور/` (يحتوي `package.json` جذري لتشغيل أوامر npm)

## 3. نموذج البيانات (Prisma)

```
Employee ──< Attendance
Employee >── WorkSchedule (shiftId اختياري)
```

| النموذج | الغرض | ملاحظات مهمة |
|---------|--------|--------------|
| `Employee` | الموظف + بصمة الوجه | `faceDescriptor Float[]` · `faceDescriptorVersion` (2=1024-d) · `hasFaceRegistered` · `emergencyCode` |
| `Attendance` | سجل يومي | فريد `(employeeId, date)` |
| `WorkSchedule` | الشفت | `startTime` · `endTime` · `lateAfter` |
| `Department` | أقسام منظمة | **منفصلة** عن `Employee.department` (نص حر) |
| `SystemUser` | مستخدمو اللوحة | مرتبط بـ Supabase Auth بالبريد |
| `Alert` | تنبيهات | بدون FK لـ Employee |

### الأدوار

| الدور | الوصف |
|-------|--------|
| `MANAGER` | صلاحيات كاملة |
| `INQUIRY_CLERK` | عرض + إضافة موظف فقط (بدون تعديل/حذف/تصدير) |

## 6. سلسلة المصادقة والصلاحيات

```
المستخدم
  → Supabase Auth (login-form.tsx)
  → middleware.ts (جلسة + حماية المسارات)
  → dashboard/layout.tsx (SystemUser نشط)
  → RoleProvider (دور في السياق)
  → الصفحات: page-auth.ts (requirePagePermission)
  → API: api-auth.ts (requirePermission)
```

### ملفات المصادقة

| الملف | الطبقة | الوظيفة |
|-------|--------|---------|
| `lib/supabase/client.ts` | عميل | متصفح |
| `lib/supabase/server.ts` | خادم | cookies |
| `middleware.ts` | Edge | `/dashboard` · `/admin` · `/login` |
| `lib/api-auth.ts` | API | `requireAuth` · `requirePermission` |
| `lib/page-auth.ts` | SSR | `getSessionSystemUser` · `requirePagePermission` |
| `lib/permissions.ts` | مشترك | تعريف الصلاحيات والأدوار |
| `lib/kiosk-auth.ts` | API كشك | `x-kiosk-key` (مقارنة timing-safe) |
| `components/dashboard/role-context.tsx` | عميل | `useUserRole` · `usePermission` |

### الكشك — مسار منفصل

- **لا يمر** عبر middleware المصادقة
- يحمى بـ `KIOSK_API_KEY` في رأس الطلب
- المفتاح يُمرَّر من صفحة الخادم إلى `KioskScanner` (مرئي في الشبكة — انظر مخالفات الأمان)

## 7. التعرف على الوجه — الارتباطات

### مصدر الحقيقة الوحيد للعتبات

`src/lib/face-match-config.ts` → `selectBestFaceMatch()`

### سلسلة الملفات

```
face-match-config.ts          ← عتبات v1/v2 + منطق الاختيار
face-engine/human-engine.ts   ← Human: كشف + بصمة 1024-d
    ├── face-recognition.ts   ← عميل: كشف · مطابقة · تسجيل
    ├── face-verify-server.ts ← خادم: مسافة · تحقق
    └── face-match-employee.ts← خادم: بحث في DB (v2)

camera-frame.ts               ← تحويل فيديو → canvas
face-descriptor-utils.ts      ← تصفية بصمات seed الوهمية

kiosk-scanner.tsx             ← حلقة المسح + تأكيد متتالي (3)
kiosk-warmup.tsx              ← تحميل مسبق للنماذج
face-enrollment-panel.tsx     ← تسجيل من لوحة التحكم
```

### العتبات الحالية (v2 — Human 1024-d)

| الثابت | القيمة | الاستخدام |
|--------|--------|-----------|
| `FACE_MATCH_THRESHOLD_V2` | 0.50 | حضور/انصراف |
| `FACE_STRONG_MATCH_DISTANCE_V2` | 0.40 | قبول مباشر |
| `FACE_MIN_GAP_FROM_SECOND_V2` | 0.08 | فرق عن وجه ثانٍ |
| `DUPLICATE_FACE_MATCH_THRESHOLD_V2` | 0.42 | منع التكرار |
| `DUPLICATE_MIN_GAP_FROM_SECOND_V2` | 0.10 | فرق تكرار |
| `CONSECUTIVE_MATCHES_REQUIRED` | 3 | لقطات متتالية في الكشك |

> v1 (128-d) عتبات منفصلة — للبيانات القديمة فقط.

### تدفق الحضور (مختصر)

```
كاميرا → detectFaceForScan → findBestMatch (عميل)
  → 3 مطابقات متتالية → GET /attendance/today
  → POST checkin/checkout + descriptor
  → verifyAttendanceFaceMatch (خادم، v2)
```
