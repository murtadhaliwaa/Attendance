# طبقات النظام — تفصيل

## الطبقة 1: العرض (UI)

### Server Components (افتراضي)
- **المسار:** `src/app/**/page.tsx`
- **الدور:** جلب بيانات أولية · تمرير props للعميل
- **أمثلة:** `dashboard/page.tsx` · `employees/page.tsx`

### Client Components
- **المسار:** `src/components/**` مع `"use client"`
- **الدور:** تفاعل · نماذج · كاميرا · تصدير
- **أمثلة:** `kiosk-scanner.tsx` · `employees-manager.tsx`

### مكوّنات UI خام
- **المسار:** `src/components/ui/`
- **الدور:** أزرار · جداول · حوارات — بدون منطق أعمال

---

## الطبقة 2: التوجيه والحماية

| الملف | النطاق |
|-------|--------|
| `middleware.ts` | جلسة Supabase · إعادة توجيه login/dashboard |
| `dashboard/layout.tsx` | SystemUser · RoleProvider · الهيكل |
| `kiosk/layout.tsx` | تحميل مسبق · غلاف الكشك |

---

## الطبقة 3: API

- **المسار:** `src/app/api/**`
- **نمط:** Handler واحد لكل HTTP method · auth أولاً · ثم lib

---

## الطبقة 4: منطق الأعمال (`src/lib/`)

مجموعات منطقية:

```
auth/          → api-auth · page-auth · permissions · kiosk-auth
employees/     → employee-* · departments · codes · validation
attendance/    → attendance-* · shifts · schedule-*
reports/       → reports · report-* · export-*
face/          → face-* · camera-frame
infra/         → prisma · supabase · query-cache · rate-limit
```

> **ملاحظة:** المجلدات الفعلية مسطّحة في `lib/` — التجميع أعلاه مفاهيمي للفهم.

---

## الطبقة 5: البيانات

## تدفق طلب نموذجي (لوحة التحكم)

```
المتصفح → GET /dashboard/employees
  → middleware (جلسة)
  → layout (SystemUser + RoleProvider)
  → page.tsx (requirePagePermission + prisma/lib)
  → EmployeesManager (client) → fetch /api/employees
  → route.ts (requirePermission + lib)
  → prisma → PostgreSQL
```

## تدفق طلب نموذجي (كشك)

```
المتصفح → /kiosk/checkin (بدون middleware auth)
  → KioskWarmup (نماذج)
  → KioskScanner → fetch /api/employees/descriptors (x-kiosk-key)
  → مطابقة محلية (Human 1024-d)
  → POST /api/attendance/checkin (تحقق خادم)
```
