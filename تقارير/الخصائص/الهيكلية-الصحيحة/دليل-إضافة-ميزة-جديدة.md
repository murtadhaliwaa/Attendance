## 2. طبقات النظام (Layered Architecture)

```
┌─────────────────────────────────────────────────────────┐
│  Presentation (عرض)                                      │
│  src/app/**/page.tsx · src/components/**                 │
│  — عرض فقط · لا منطق أعمال معقد · لا استعلام DB مباشر   │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  API / Server Actions                                    │
│  src/app/api/**/route.ts                                 │
│  — requirePermission() أو requireKioskAuth()             │
│  — التحقق من المدخلات · استدعاء lib · JSON response    │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  Business Logic (منطق الأعمال)                           │
│  src/lib/**                                              │
│  — دوال خالصة · قابلة للاختبار · بدون React             │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  Data Access                                             │
│  prisma · src/lib/prisma.ts                              │
└─────────────────────────────────────────────────────────┘
```

### قواعد الطبقات

| الطبقة | يفعل | لا يفعل |
|--------|------|---------|
| **Page (SSR)** | `getSessionSystemUser` · `requirePagePermission` · استدعاء lib | `fetch` لـ API داخلي · منطق وجه |
| **Client Component** | حالة UI · `fetch` API · hooks | استعلام Prisma · مفاتيح سرية |
| **API Route** | auth guard · validation · lib · status codes | منطق 200+ سطر — انقله لـ lib |
| **lib/** | كل المنطق القابل لإعادة الاستخدام | `"use client"` · JSX |

## 3. قالب إضافة ميزة — خطوة بخطوة

### أ) ميزة في لوحة التحكم (مثال: «إجازات»)

```
1. prisma/schema.prisma        → نموذج Leave + علاقات
2. npx prisma db push          → تطبيق (أو migration لاحقاً)
3. src/lib/permissions.ts      → leave:read · leave:write
4. src/lib/leaves.ts           → getLeaves · createLeave · validate
5. src/app/api/leaves/route.ts → GET/POST + requirePermission
6. src/app/dashboard/leaves/page.tsx → SSR + requirePagePermission
7. src/components/dashboard/leaves/leaves-manager.tsx → "use client"
8. src/lib/dashboard-nav.ts    → عنصر قائمة (إن لزم)
9. تقارير/الخصائص/تطويرات/09-الإجازات.md → توثيق
```

### ب) ميزة في الكشك

```
1. src/lib/kiosk-types.ts      → تسميات عربية إن لزم
2. src/app/api/.../route.ts    → requireKioskAuth(request) أول سطر
3. src/components/kiosk/...    → واجهة العميل
4. لا تمر عبر middleware المصادقة — استخدم KIOSK_API_KEY فقط
```

### ج) ميزة تلمس الوجه

```
1. عدّل العتبات في face-match-config.ts فقط — لا تكرر قيم
2. عميل: face-recognition.ts
3. خادم: face-match-employee.ts أو face-verify-server.ts
4. وثّق العتبة الجديدة في تطويرات/02-التعرف-على-الوجه.md
```

## 4. هيكل الملفات الموصى به

### مكوّن جديد

```

```

```
src/app/api/[feature]/route.ts
src/app/api/[feature]/[id]/route.ts
```

## 5. الأمان — إلزامي لكل API

```typescript
// لوحة التحكم
const auth = await requirePermission("employees:read");
if (auth.error) return auth.error;

// الكشك
const kioskError = requireKioskAuth(request);
if (kioskError) return kioskError;

// التحقق من المدخلات
if (!isValidFaceDescriptor(descriptor)) {
  return NextResponse.json({ error: "..." }, { status: 400 });
}
```

### قواعد أمان

| القاعدة | السبب |
|---------|--------|
| لا `SUPABASE_SERVICE_ROLE` في العميل | تسريب كامل للقاعدة |
| لا `KIOSK_API_KEY` في كود عام | يظهر في bundle — قلل الاعتماد عليه للقراءة الحساسة |
| تحقق الخادم دائماً من الوجه | العميل قابل للتلاعب |
| `select` محدود في Prisma | لا تُرجع `faceDescriptor` إلا عند الحاجة |
| Rate limit للمسارات العامة | مثل `emergency/route.ts` |

## 6. الصلاحيات — النمط المعتمد

```typescript
// 1. عرّف في permissions.ts
export type Permission = ... | "leaves:read" | "leaves:write";

// 2. أضف للأدوار في ROLE_PERMISSIONS

// 3. API
await requirePermission("leaves:read");

// 4. صفحة SSR
await requirePagePermission("leaves:read");

// 5. واجهة عميل
const canWrite = usePermission("leaves:write");
```

## 7. العميل vs الخادم

| `"use client"` | Server Component / API |
|----------------|------------------------|
| `useState` · `useEffect` · كاميرا | `prisma` · `createClient` server |
| `face-verify-server` | `computeFaceMatchDistance` · `verifyFaceDescriptor` |
| `usePermission` | `requirePermission` |
| تصدير Excel/PDF | تقارير JSON من API |

**لا تستورد** `prisma` أو `@/lib/supabase/server` من مكوّن عميل.

## 8. التسمية والاتفاقيات

| العنصر | الاتفاق |
|--------|---------|
| ملفات lib | `kebab-case.ts` |
| مكوّنات | `PascalCase.tsx` |
| API routes | مجلد باسم المورد · `route.ts` |
| صلاحيات | `resource:action` مثل `employees:update` |
| رسائل خطأ API | عربية · `{ error: string }` |
| التواريخ | `Asia/Riyadh` عبر `app-timezone.ts` |
