# تكرار الكود والتجزئة

## 1. المصادقة — ثلاث نقاط تحقق

| الموقع | ماذا يفعل |
|--------|-----------|
| `middleware.ts` | Supabase session → redirect |
| `dashboard/layout.tsx` | Supabase + SystemUser → redirect |
| `page-auth.ts` / `api-auth.ts` | نفس المنطق للصفحات/API |

**المشكلة:** تغيير قاعدة الدخول يتطلب تعديل 3 أماكن.  
**الحل المقترح:** دالة `resolveSessionUser()` موحّدة في `lib/session.ts`.

---

## 2. التعرف على الوجه — عميل vs خادم

| المنطق | عميل | خادم |
|--------|------|------|
| مسافة إقليدية | face-api.js | حلقة يدوية في face-verify-server |
| اختيار أفضل مطابقة | `selectBestFaceMatch` + gap | `verifyFaceDescriptor` **بدون gap** |
| منع التكرار | `findStrictDuplicateMatch` | `findEmployeeByFaceDescriptor` |

**المشكلة:** checkin/checkout لا يطبقان gap — احتمال قبول خاطئ نظري.  
**الحل المقترح:** `verifyFaceMatchWithAmbiguity(stored, provided, allEmployees)` في الخادم.

---

## 3. إعادة تصدير العتبات

```
face-match-config.ts
  → face-recognition.ts (re-export كبير)
  → face-verify-server.ts (re-export كبير)
```

**المشكلة:** سلاسل import مربكة · aliases قديمة `MATCH_THRESHOLD`.  
**الحل المقترح:** استيراد مباشر من `face-match-config` فقط · حذف aliases.

## 4. نظامان للأقسام

| النظام | الموقع |
|--------|--------|
| `Department` model | جدول منظم · إعدادات |
| `Employee.department` | نص حر على الموظف |

**المشكلة:** عدم اتساق — موظف كشك يُنشأ بـ `"عام"`.  
**الحل المقترح:** FK لـ Department أو مزامنة إلزامية.
