# 02 — التعرف على الوجه

## الوصف
كشف الوجه ومطابقته وتسجيل البصمة باستخدام **@vladmandic/human** (عميل) مع تحقق خادم. البصمة الحالية **1024-d** (إصدار v2).

## مصدر الحقيقة
**`src/lib/face-match-config.ts`** — لا تعدّل العتبات في مكان آخر.

## الملفات

| الملف | الدور |
|-------|-------|
| `face-engine/human-engine.ts` | كشف + بصمة 1024-d (Human) |
| `face-engine/constants.ts` | `HUMAN_MODEL_BASE` → `/models/human/` |
| `face-match-config.ts` | عتبات v1/v2 + `selectBestFaceMatch()` |
| `face-descriptor-version.ts` | v1=128-d · v2=1024-d |
| `face-match-distance.ts` | مسافة Human (خادم + عميل) |
| `face-recognition.ts` | واجهة عميل: تحميل · كشف · مطابقة · تسجيل |
| `face-verify-server.ts` | تحقق خادم |
| `face-match-employee.ts` | بحث في DB (v2 فقط للكiosk) |
| `face-descriptor-utils.ts` | تصفية بصمات وهمية · `needsFaceReEnrollment` |
| `camera-frame.ts` | إطار فيديو للكشف |
| `public/models/human/*` | نماذج Human المحلية |

## نماذج Human

| النموذج | الاستخدام |
|---------|-----------|
| blazeface | كشف الوجه |
| facemesh | معالم الوجه |
| faceres | بصمة **1024-d** |

تحميل: `npm run models:download` (ينسخ من `node_modules/@vladmandic/human/models`)

## أوضاع المطابقة (v2)

| الوضع | الثابت | الاستخدام |
|-------|--------|-----------|
| `recognize` | `FACE_MATCH_THRESHOLD_V2` 0.50 | حضور/انصراف |
| `duplicate` | `DUPLICATE_FACE_MATCH_THRESHOLD_V2` 0.42 | منع تسجيل مكرر |

> v1 (128-d) محفوظ للبيانات القديمة فقط — الكiosk يستخدم v2 حصرياً.

## التدفقات

### مسح حضور (كشك)
```
detectFaceForScan (Human) → findBestMatch(recognize)
→ 3 مطابقات متتالية → POST checkin + descriptor
→ verifyAttendanceFaceMatch (خادم، v2)
```

### تسجيل (لوحة)
```
captureEnrollmentDescriptor (7 عينات)
→ findEmployeeByFaceDescriptor (duplicate, v2)
→ حفظ عبر POST/PUT employees + faceDescriptorVersion=2
```

### تسجيل (كشك — موظف جديد)
```
captureEnrollmentDescriptor
→ findStrictDuplicateMatch (عميل)
→ PUT /api/employees/descriptors (خادم duplicate)
```

انظر `face-match-config.ts` أو `00-مرجع-النظام-الشامل.md` §7.
