# 05 — الحضور والانصراف

## الوصف
تسجيل check-in/check-out يومي لكل موظف مع حالة (حاضر · متأخر · انصراف مبكر).

## API

| Method | المسار | الملف |
|--------|--------|-------|
| POST | `/api/attendance/checkin` | `checkin/route.ts` |
| POST | `/api/attendance/checkout` | `checkout/route.ts` |
| POST | `/api/attendance/emergency` | `emergency/route.ts` |
| GET | `/api/attendance/today` | `today/route.ts` |

## منطق الأعمال

| الملف | الدور |
|-------|-------|
| `attendance-utils.ts` | حساب LATE · EARLY_LEAVE |
| `attendance-shift.ts` | حل شفت الموظف |
| `attendance-reconcile.ts` | مزامنة حالات |
| `app-timezone.ts` | Asia/Riyadh |

## نموذج Attendance

- فريد: `(employeeId, date)`
- `method`: FACE | EMERGENCY_CODE
- `status`: PRESENT | LATE | EARLY_LEAVE | ABSENT

## تدفق checkin

```
1. requireKioskAuth
2. isValidFaceDescriptor
3. جلب موظف نشط + وجه مسجّل
4. verifyFaceDescriptor
5. upsert attendance لليوم
6. حساب status من الشفت
```

## تدفق emergency

- Rate limit: 5 طلبات/دقيقة/IP
- مطابقة `emergencyCode` بدون وجه

## العرض

- `dashboard/page.tsx` — ملخص اليوم
- `kiosk-scanner.tsx` — تسجيل مباشر
