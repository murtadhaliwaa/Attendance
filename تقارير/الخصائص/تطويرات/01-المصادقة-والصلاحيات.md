# 01 — المصادقة والصلاحيات

## الوصف
تسجيل دخول المدراء وموظفي الاستعلامات عبر Supabase، مع ربط بـ `SystemUser` في Prisma للصلاحيات.

## الملفات

| الملف | الطبقة |
|-------|--------|
| `src/middleware.ts` | Edge — حماية المسارات |
| `src/lib/supabase/client.ts` | عميل |
| `src/lib/supabase/server.ts` | خادم |
| `src/lib/api-auth.ts` | API guards |
| `src/lib/page-auth.ts` | SSR guards |
| `src/lib/permissions.ts` | تعريف الأدوار |
| `src/lib/kiosk-auth.ts` | مفتاح الكشك (منفصل) |
| `src/components/auth/login-form.tsx` | نموذج الدخول |
| `src/components/dashboard/role-context.tsx` | سياق الدور |
| `src/app/dashboard/layout.tsx` | بوابة اللوحة |
| `src/app/login/page.tsx` | صفحة الدخول |

## التدفق

```
POST login (Supabase) → cookie session
→ middleware يتحقق من الجلسة
→ layout يتحقق من SystemUser.isActive
→ RoleProvider يوفّر الدور للـ UI
→ API: requirePermission
```

## الحسابات الافتراضية
- `hr@company.com` — MANAGER
- `inquiry@company.com` — INQUIRY_CLERK
- إعداد: `npm run auth:setup-production`
